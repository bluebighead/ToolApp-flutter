# 心率设备连接记忆功能设计文档

## 概述

为心率广播接收器添加设备连接记忆功能，使用 SharedPreferences 持久化上次成功连接的设备信息，实现下次打开时自动连接。

## 需求

1. 用户首次手动连接成功后，保存设备ID和名称到本地存储
2. 下次打开页面点"开始接收"时，自动尝试连接记忆的设备
3. 如果设备未开机，持续扫描等待，设备出现后自动连接
4. 用户手动选择其他设备时，更新记忆为新设备
5. 用户断开连接后，记忆保留（不清除）
6. 设备列表中标记"上次连接"的设备，方便识别

## 技术方案

### 存储层
- 复用项目已有的 `shared_preferences` 依赖
- 在 `HeartRateBle` 类中封装记忆功能，不暴露给页面层
- 存储键：`last_connected_ble_device_id`、`last_connected_ble_device_name`

### 自动连接流程
1. `_startBle()` 检查是否有记忆设备
2. 有记忆设备 → 开始扫描，在扫描回调中匹配设备ID
3. 匹配成功 → 自动调用 `connectToDevice`，停止扫描
4. 匹配失败 → 继续扫描，状态显示"正在寻找上次连接的设备..."
5. 无记忆设备 → 保持原有行为（扫描等待用户选择）

### 记忆更新时机
- 用户手动点击"连接"按钮成功连接后
- 自动连接成功后
- 用户断开连接时不清除记忆

### UI变化
- 设备列表中上次连接的设备显示"上次连接"标签
- 状态栏显示"正在寻找上次连接的设备..."提示
- 自动连接过程中"开始接收"按钮变为"停止扫描"

| 文件 | 变更类型 | 说明 |

| `lib/utils/heart_rate_ble.dart` | 修改 | 添加 SharedPreferences 记忆功能、自动连接方法 |
